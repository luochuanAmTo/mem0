 Mem0 系统中所有具体“记忆”实现类的“设计蓝图”。无论底层是将记忆存储在向量数据库、传统数据库还是其他地方，对于上层应用来说，它们都必须提供 `get`, `get_all`, `update`, `delete`, `history` 这五种标准操作方法



| 方法                      | 作用                                                         |
| :------------------------ | :----------------------------------------------------------- |
| `get(memory_id)`          | **精确检索**：根据唯一的 `memory_id`，获取一条具体的记忆内容。 |
| `get_all()`               | **批量获取**：列出当前范围内（例如某个用户或智能体）存储的所有记忆。 |
| `update(memory_id, data)` | **内容更新**：用新的 `data` 替换指定 `memory_id` 的旧记忆内容。 |
| `delete(memory_id)`       | **删除记忆**：根据 `memory_id` 移除一条不再需要的记忆。      |
| `history(memory_id)`      | **变更审计**：查看某条记忆的历史修改记录，实现记忆的可追溯性。 |

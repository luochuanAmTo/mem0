# mem0 在解决什么问题

**给 AI 加“长期记忆”能力**

传统 LLM：只能靠 prompt（上下文窗口），聊完就忘，成本高（token 爆炸）

mem0 做的事：自动从对话中**提取“重要信息”**，存下来（memory），下次用时**检索 relevant memory**

核心流程：

```
输入对话 → 提取记忆 → 存储 → 检索 → 注入上下文
```



对应论文，mem0实现了，基本memory读写操作，

​                    memory 抽取（LLM infer），LLM 抽取的是 **自然语言形式的事实/偏好/指令**存储为一个**结构化的记忆对象**（含记忆文本 + 元数据）**不强制要求抽取成三元组**，不形成图

​           	memory存储（ 向量嵌入），元数据（user_id, agent_id, session 等）底层依赖 **向量数据库 + 过滤**

​                    memory retravial :对用户查询做 embedding,在向量空间中做 **语义相似度搜索**，通过元数据做进一步过滤

​                    memory revolution：**记忆记录级更新**，更新一个完整记忆对象（事实文本 + 元数据），不暴露节点和边的概念给用户

未完全实现： memory 抽取：从文本中 **抽取结构化三元组**构建图，支持图推理，

​			 memory 存储，论文中：知识图谱 / 时序图 / 超图 / 层次图 / 混合图存储内容是 **实体节点 + 关系边**目标是支持 **graph traversal、relationship reasoning**

​			memory retravial:论文中**Graph-based retrieval**：在图结构上游走、查找推理路径支持 **结构化推理路径返回**（例如症状演化链、家族史关联）

​                        memory 演化（更新 / 冲突处理）论文中**节点 / 边级编辑**例如：将 `(胃痛, 状态, 活跃)` 改为 `(胃痛, 状态, 已解决)`
